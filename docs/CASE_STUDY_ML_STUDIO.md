# ML Studio Demo Case Study

## Problem

Manufacturing analytics teams need to monitor equipment telemetry, generate model-ready features, run anomaly and supervised learning workflows, and inspect model outputs without switching between disconnected tools.

## Goal

Create a dashboard-centered ML studio that connects operational telemetry, model execution, model result review, threshold alerts, and exploratory analysis.

## Composition

- React dashboard for operators and analysts
- Spring Boot API for data access and workflow orchestration
- FastAPI AI service for model execution
- MongoDB collections for telemetry, features, policies, and results

## Features

- AI operation status monitoring
- Dataset and feature configuration
- Anomaly detection result review
- Supervised model result analysis
- Threshold alert monitoring
- Model training policy management
- Data exploration views

## Outcome

The demo shows a reusable full-stack pattern for ML operations dashboards while avoiding real customer names, company names, facility names, equipment names, and operational records.

## Expansion Ideas

- Add a one-command demo seed script
